21 /****************************************************************************
22  * OrderWindow Class
23  * A QDialog type class used to create the OrderWindow. The order
24  * window has 4 sections of display:
25  * 1. The three iRobot pictures, names, prices, comboboxes for plan
26  * option prices, quantity spinboxes, and subtotal for each iRobot.
27  * 2. The price breakdown including subtotal, shipping & handling
28  * charges, tax amount, and total price for the order.
29  * 3. Billing and shipping address
30  * 4. Credit card information
31  *
32  * A purchase button becomes active when at least one iRobot is ordered.
33  * When the purchase button is clicked, several conditions are checked
34  * and if it passes all of them, a confirmation messagebox is
35  * opened. Closing the confirmation box also closes the OrderWindow
36  * and returns active control to the ProductWindow. When the cancel
37  * button is clicked, active control is returned to the ProductWindow.
38  *
39  * Conditons that are checked before saving the order to the sqlite
40  * database file includes:
41  * ccName (orders table) = Company (in the customers table)
42  *
43  * PRE-CONDITONS
44  * The following constant variables must be declared and initialized
45  * in the productWindow.h.
46  * ROBOT_A_NAME : name of robot A
47  * ROBOT_A_PRICE : price of robot A
48  * ROBOT_B_NAME : name of robot B
49  * ROBOT_B_PRICE : price of robot B
50  * ROBOT_C_NAME : name of robot C
